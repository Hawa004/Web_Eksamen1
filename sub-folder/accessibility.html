Accessibility

Key changes

Design and functionality: We replaced the video with a picture as it was more fitting. To be more functional, we added the subject name at the top, and the bachelor program under the subject name for easier recognition of the program.

Color: The original color scheme is bright yellow, which represents OsloMet. We opted for a softer color palette to balance out the contrast. This ensures better readability of our website, especially for users sensitive to bright colors.

Length: The original webpage was too long as all the information boxes were placed vertically. By changing the structure to horizontal it allows the user to get all the information in one main table, without having to scroll back and forth. This can be helpful to attain information more effectively.

Testing tool

To evaluate the accessibility of our website we used both manual and automated testing tools, such as Lighthouse and live server by Visual Studio Code. After fishing the website, we tested it with an automated testing tool, Lighthouse, which can help us to improve the quality of our website. The test revealed a score of 73 on accessibility, which indicates some room for improvement.

Two categories of improvements

1. Name and labels

Buttons In this area the report detected several buttons that do not have an accessible name. This makes it difficult for screen readers to understand the purpose of the button.

Changes made: We added the button element in the html, and aria-label since we are using an icon.

2.Table and lists

Lists do not contain only list elements and script supporting elements

The report identifies that some of the ul elements contain other elements that are not list , like "strong" and script".

Changes made:For semantic reasons we can make sure that "ul" contains "li" elements. Instead of nesting "strong" inside "li", we can style it with CSS.

List elements "li" are not contained within "ul", "ol" or "menu" parent elements.

In this area the report identified that there exist "li" items that are not properly nested within the parent element, this is rather challenging for screen readers to understand as it disturbs the sematic structure of the content.

Changes made: Nest all the items within the parent list. Move script before ul. We also made changes to the CSS so we can get the look we wanted.
