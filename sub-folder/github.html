Github-page

How working with Github for the first has been:

In the beginning, we committed every few days with larger commits, often using a single commit message that didn’t specify all the changes that were done. Over time, we learned that small and frequent commits with descriptive commit messages made it easier to track changes in detail. This approach allowed us to easily identify where things went wrong and efficiently pinpoint the bugs when they arose. Additionally, this method ensures accountability, which again made it easier for us to maintain a clean main branch when merged carefully as well.

Using branches in GitHub:

At the start, we worked in separate branches as we distributed different tasks among the group members. This allowed us to work independently on different parts of the project without directly affecting the main branch. Through this approach, we developed a strong understanding of how to use Git actively. While we were working individually in our own branches, we also gained hands-on experience in creating HTML and CSS. Another feature we found particularly useful was pull requests, as they allowed us to discuss and review proposed changes as a team before merging them into the main branch. This approach improved not only the communication among the group but also ensured that the quality of the code still remained the same.
