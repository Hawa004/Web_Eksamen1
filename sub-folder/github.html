How working with Github for the first has been:

In the beginning, as you might notice ,we used to commit every few days with larger commits, often using a single commit message that didn’t specify all the changes that were done. Overtime, we learned that small and frequent commits with descriptive commit messages made it easier to track detailed history . With this approach we could easily identify were things eventually could go wrong and pinpoint the bugs when they arose. Additionaly this ensures accountability, which again makes it easier for us to maintain a clean main branch when merged carefully as well.

Using branches in GitHub

At the very beginning, we used to work in separate branches as we had distributed different tasks to each member of the group. This allowed us to work on different parts of the project without directly affecting the main branch. Through this approach , we gained an understanding of how to use Git actively, even though we were working individually in our own branches, while at the same time each individual became familiar with creating Html and css. Another thing we found particularly useful was pull requests, as they allowed us to discuss and review proposed changes before merging them into the main branch.
